Git仓库推送指南

将本地仓库内容推送到远程仓库的完整流程

步骤1: 初始化本地Git仓库

git init

在当前项目目录执行此命令初始化Git仓库

步骤2: 创建Python项目.gitignore文件

echo "__pycache__/" > .gitignore
echo "*.py[cod]" >> .gitignore
echo "venv/" >> .gitignore
echo ".env" >> .gitignore

为Python项目添加必要的忽略规则，避免提交不必要的文件

步骤3: 添加文件到暂存区

git add .

添加所有文件到Git暂存区，准备提交

步骤4: 提交更改

git commit -m "初始提交"

创建第一个提交，记录当前项目状态

步骤5: 关联远程仓库

git remote add origin 您的远程仓库URL

将本地仓库与远程仓库关联，替换"您的远程仓库URL"为实际URL

步骤6: 推送更改到远程仓库

git push -u origin main

将本地提交推送到远程仓库的主分支

如果是第一次推送，可能需要使用 git push -u origin main --force 强制推送

状态检查命令

git status
git remote -v
git log --oneline

这些命令可以帮助您检查当前Git仓库的状态
